## 1. Why (the wall WS1/WS2 hit)

WS1's fret-map calibration lifted ambiguous-note string accuracy 0.544 → 0.574 and
is fusion-relevant, but ~**68% of ambiguous notes are on clips where YOLO detects
~0 fret OBBs at conf=0.25** — geometry physically cannot move them. A learned model
resolves the string **directly from pixels** (the fretting-hand on the neck),
bypassing both the fret-detection bottleneck and the fragile fingertip→string
projection. This is the highest-ceiling option; the cost that gated it (training
data + GPU + money) is now accepted.

## 2. Goal & exit criteria

Frame of measurement unchanged (gold-audio, the "0.94 frame"). Eval on the **held-out
GAPS `test` split** (clean-12 ⊂ test — no leakage).

| Criterion | Now (geometry) | WS4 target |
|---|---:|---:|
| Leading: ambiguous-note string accuracy (test) | 0.574 | **beat it decisively** (→ 0.75+) |
| Lagging: gold `+real(auto)` Tab F1, no per-clip regression | 0.8148 (gated no-op) | **> 0.865**, materialised |
| Stretch | — | toward the 0.94 single-line bar (oracle 0.9726) |

Go/no-go after the first trained checkpoint: if it doesn't beat the geometric 0.574
on held-out test, stop and reassess.

## 3. Data (the assessment that cleared the STOP)

- GAPS metadata (`gaps_metadata_with_splits.csv`): **270 `train` / 30 `test` / 101
  unassigned** clips, each with **gold tab** (free string/fret labels), a **`yt_id`**
  (downloadable), and **syncpoints** (alignment provided).
- Trainable labels: **~216K ambiguous notes across the train split** (240K total notes
  over 396 parsed clips; ~140K+ in `train`). Labels = gold `string_idx` per note.
- **No eval leakage:** clean-12 ⊂ the `test` split, held out. Train on `train` only.
- **License:** GAPS is NC → accepted for this non-commercial project; the resulting
  weights are an **NC-tainted artifact** (record in LICENSES.md). The crop-extraction
  uses the AGPL ultralytics YOLO (already a project taint).

## 4. Architecture

**Model:** a pretrained image backbone (torchvision `resnet18`/`mobilenet_v3` or
`timm`) → a small head emitting **6-way string logits** (`cfg.n_strings`).

**Input:** the **YOLO neck-region crop** at the note's onset frame, resized to a fixed
size. Using the raw neck crop (not a homography-rectified board) lets the CNN learn
pose/perspective invariance and **skips the geometry entirely** — the whole point of
WS4. Only the reliable YOLO `neck` box is needed (no MediaPipe, no fret OBBs, no
homography).

**Conditioning on pitch:** the model outputs an unconditional 6-way string posterior;
at fusion the known pitch restricts to its candidate strings (mask + renormalise).
Trained with the gold string as the target (cross-entropy).

**Fusion integration (no §8 change):** the per-note string posterior `p(s)` →
a `(n_strings, max_fret+1)` matrix placing `p(s)` on each candidate cell
`(s, fret = fret_of(pitch, s))` → fed through the **existing
`marginal_string_fret` → `AudioEvent.fret_prior`** channel, exactly like the
geometric evidence. Absent/low-confidence → audio-only (no-regression invariant holds).

## 5. Pipeline

1. **Acquire (done):** `scripts/acquire/gaps_video.py --download --clips train`
   (failure-tolerant; attrition logged). Running.
2. **Extract** (`scripts/train/extract_string_dataset.py`, new): per train clip,
   align (offset cache / syncpoints), and for each gold note grab the onset frame,
   crop the YOLO neck region, save `(crop.jpg, string_idx, pitch_midi, fret, stem)`.
   Incremental + resumable; writes a manifest (parquet/jsonl). Filter low-alignment
   clips to limit label noise.
3. **Train** (`scripts/train/string_resolver_modal.py`, new): mirror
   `yolo_guitar_obb_modal.py` — a Modal Volume holds the extracted crops + manifest;
   an `@app.function(gpu="L4")` fine-tunes the backbone and returns the weights tar.
   Needs `pip install modal` + the user's Modal auth.
4. **Eval** (`scripts/eval/v1_1_gaps_video_chain_probe.py` + diag, new `--learned`
   path): load the checkpoint, produce the per-note string posterior, fuse via
   `fret_prior`, score on held-out clean-12/test. Compare to geometric 0.574 / audio
   0.8148.

## 6. Contracts & immutability (SPEC §8)

Stays implementation-side: the learned evidence rides the existing
`marginal_string_fret`/`fret_prior` channel; a learned `HandBackend`-style component
returns a per-note string posterior. No dataclass/Protocol/entrypoint signature
change. New deps (`torch`/`torchvision`/`timm`, `modal`) are training/eval-only — the
runtime inference path can load just the checkpoint (torch).

## 7. Risks

- **Label noise from alignment** — lower-coverage train clips → some onset frames land
  off; mitigate by filtering on alignment `peak_ratio` + using syncpoints; the eval is
  on the clean test split.
- **Download attrition** — dead/geo-blocked uploads reduce the effective train set;
  logged, tolerated (270 is ample headroom).
- **Transfer** — in-domain (GAPS) should work; generalisation beyond GAPS-like footage
  is a v1.1-video question, acceptable for this scope. The chunk-5 transfer caution is
  largely mitigated by training *in-domain*.
- **NC taint / AGPL** — accepted; recorded.
- **Modal auth** — the GPU run needs `modal` installed + the user's token; the
  extraction is local (CPU YOLO, slow) or could run on Modal too.

## 8. Decisions — CONFIRMED 2026-06-25

1. **Full 270-clip train run** (not a pilot) — user choice.
2. **Modal** GPU (reuse the YOLO-OBB infra pattern).
3. **NC license accepted** (non-commercial project).
4. **Input = raw YOLO neck-crop**, output = 6-way string posterior, pitch-conditioned
   at fusion, evidence via `fret_prior` (no §8 change) — design choice, revisitable if
   the first checkpoint underperforms.
